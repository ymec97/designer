## Architecture rules that bite (from CLAUDE.md — repeated because they gate this work)

- **All board mutations flow through `BoardDocument.perform(_:actionName:)`** —
  one undoable `BoardOperation` per user action; `Board.apply()` returns the
  inverse. The canvas never mutates directly; it calls its delegate.
- **Tolerant coding:** model types keep unknown JSON keys in `extra` bags.
  New persisted fields must be added to `CodingKeys` + `init(from:)` /
  `encode(to:)` or they silently drop. (Relevant to Workstream C's `textSize`.)
- **Agent wire format is name-addressed and lossy on purpose** (`WireBoard`):
  node `extra`, ink, boundaries never cross it. Proposals parse anchored to
  the current board — matched nodes inherit position/style for unset fields.
  If you add a model field (e.g. `textSize`), **decide explicitly** whether
  agents may see/set it.
- **Style:** hex `#RRGGBB[AA]`, `fill: "none"` sentinel (`Style.noFill`),
  element `opacity` 0…1. Nodes, ink, notes, edges all carry a `Style`.
- **Coordinates:** world space is y-down; `CanvasViewport` (origin+scale)
  maps world↔view. Never size UI chrome from zoomed view rects without
  clamping.
- **Performance is priority #1** (120 Hz): the canvas has a far-zoom batch
  fast path that bypasses per-node drawing. Per-node decorations must live in
  an overlay pass or accept being skipped at far zoom. Caches rebuild on
  `board` didSet, never per frame.

## Progress so far

### Workstream D — undo integrity (IN PROGRESS)

Committed in this branch (compiles clean via `swift build`; **not yet
battery-tested**):

- **`DesignerModel/BoardOperation.swift`** — added `var isNoOp: Bool` (empty
  batch, or a batch whose children are all no-ops).
- **`Designer/BoardDocument.swift`**:
  - Added an `undoManager` override that sets `groupsByEvent = false` **once**
    (in the setter), instead of on every `perform`. *Root-cause fix:* setting
    `groupsByEvent = false` on every `perform` stranded AppKit's already-open
    automatic per-event top-level group (its runloop-end close observer stops
    firing), so after a couple of edits `undo()` silently no-ops — this is the
    "⌘Z gets stuck" bug.
  - `perform` now `guard !operation.isNoOp else { return }` at the top (no
    dead undo steps), and no longer touches `groupsByEvent`.

linked-board view is showing. Inside a linked board, ⌘Z currently mutates the

## Workstreams (the full plan)

### A — Canvas rendering & recognition
Files: `DesignerCanvas/BoardRenderer.swift`, `DesignerCanvas/CanvasView.swift`,
`DesignerRecognition/StrokeRecognizer.swift`, `DesignerModel/EdgeGeometry.swift`

- **Remove the kind-dot entirely.** Delete the dot block in `drawNode`
  (`BoardRenderer.swift` ~:301-322); the far-zoom batch path never drew it, so
  removal is local. Kind still drives the default tint.
- **Focus-mode invisible text.** Dimming is one `context.setAlpha(0.22)`
  (`dimmedAlpha` / `withFocusAlpha` ~`CanvasView.swift:2387`) over the whole
  element; the gray `#8B95A5` fill (luminance ≈0.58) makes `textColor(onFill:)`
  (`BoardRenderer.swift:439`, cutoff 0.55) pick DARK text, which at 22% over
  the graphite ground vanishes. Fix: render the label at a higher floor alpha
  than the fill when dimmed, or pick the label color against the
  dimmed-composited fill. Verify on `Webapp` / `Astelia State` with focus on
  another layer.
- **Connector caption flicker while zooming.** `CaptionPlacer` is rebuilt
  every frame (`beginCaptionPass` ~:52); `place()` (`EdgeGeometry` ~:315) is a
  greedy first-fit whose `pillSize` is fed in WORLD units (`pillView/scale`),
  so as scale changes mid-zoom the chosen slot jumps frame to frame. Fix:
  persist each edge's chosen caption fraction and reuse it while a viewport
  change/animation is in flight; re-solve only when settled.
- **Weird edge routing.** Reproduce on the fixture. Residual hooked/crossing
  cases (from `route()` ~:154-190): when `sideFaces` fails the endpoint keeps
  the straight-line anchor while the curve arrives from the detour side (hook);
  the `>220pt` cluster-skip (~:294) lets the route cross that block; the
  two-fail fallback returns a straight line through blockers (~:185);
  `anchorSpread` (~:445) only co-plans pure-auto sibling edges so re-anchored
  ones still converge. Tighten: re-validate the final smoothed spline against
  the target's own border and prefer the facing side; co-plan spread with the
  re-anchor. Test with **real mouse drags** moving nodes, asserting arrivals
  stay on a facing border with no self/blocker intersection.
- **Circle → rectangle misrecognition.** `recognizeClosedShape`
  (`StrokeRecognizer.swift` ~:75-127): ellipse gate is `circularity > 0.82`;
  jittery circles land ≤0.82 and fall through to corner-counting → rectangle.
  Loosen the ellipse gate and/or add a roundness pre-check before corner
  counting; add regression cases with wobbly-circle point sets.
- **Text-size rendering hook.** Replace the hardcoded base label sizes (node
  `13*scale` ~`BoardRenderer.swift:356`, note `12*scale` ~:482) with a
  per-element size (Workstream C's field) × scale, clamped so the label never
  exceeds the element frame.
