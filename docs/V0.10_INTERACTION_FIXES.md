# v0.10 interaction fixes — context, plan & test plan

Branch: `feature/v0.10-interaction-fixes` (off the unified `feature/v0.10-feedback`).
Authored in a Linux cloud session with **no Swift/Xcode toolchain** — nothing here
is compile-verified. Run the battery on a Mac (see below) before merging.

## Context

Five interaction regressions/requests surfaced after the v0.10 merge, mostly
interactions *between* recent changes (F11 no-fill hit-testing, B13 ink move, the
merged connector-clutter anchor-slots). All fixes are localized to
`DesignerCanvas/CanvasView.swift` + `BoardRenderer.swift`.

## Items, root cause, and fix

**I1 — No-fill rectangle: border selects & moves (not connect).**
F11 made a no-fill node hittable only on its ~8px border, but that border is also
the 10px connect band, so every border mousedown became a `.connect` and the rect
could never be selected/moved. Fix: gate connector-initiation (and the connect
crosshair) on `hasFill || image` — no-fill outlines fall through to select + move;
real connections from solid blocks are unchanged. (`mouseDown` connect-band gate;
`updateCursor`.)

**I2 — Text-box resize scales the text.**
A standalone text box is a `.note`; resizing set only `note.frame`, and the font
was a fixed `12 × textSize × scale`. Fix: a note's font now tracks its box HEIGHT
(`frame.height * 0.55 × effectiveTextMultiplier`, still width-clamped by
`clampedLabelFontSize`), so drag-resizing the box scales the text. (`drawNote`.)

**I3 — Borderless caret-only text-box editing + live size.**
The editor `NSTextField` drew a solid background box and a fixed font. Fix (notes
only): `drawsBackground=false`, `focusRingType=.none` (caret only, no box); the
editor font matches the note's computed size and refreshes live on `board` didSet
(`refreshLabelEditorFontIfEditing`) so size changes show while editing; the note
selection ring is dropped so an unselected/edited text box shows no outline.

**I4 — Moving a drawing shows the stroke, not just the guides.**
The B13 live-ink translation only ran in the far-zoom branch; at edit zoom the
`drawables` loop passed `frameOverride`, but `renderer.draw`'s `.ink` case ignored
it, so the stroke stayed at its origin (only snap guides moved). Fix: honor
`frameOverride` in `renderer.draw`'s `.ink` case (translate points by the transient
bbox delta); the far-zoom pass was deduped to the same path.

**I5 — Connector endpoint snap: weaker + hysteretic.**
Endpoint targeting used `editableNode` (plain `frame.contains`, fill-blind, no
hysteresis), so a big no-fill grouping rect was an always-on magnet and the target
flipped between neighbours every mouse-move. Fix: target with the fill-aware
`editableElement`/`preciseHit` (no-fill rects hollow → border-only), and hold the
previous SOLID target while the cursor stays within its frame + a small margin.

## Automated tests added (`Designer/UITest.swift`, run via `--ui-test`)

- **step33NoFillRectBorderMovesNotConnect** (I1): mousedown+drag on a no-fill
  rect's border moves it (~drag delta) and creates NO connector.
- **step34InkDragShowsStrokeMoving** (I4): renders mid-drag and asserts the ink
  stroke draws at the moved position (isolated scene → any non-bg pixel near the
  dragged point is the stroke), then asserts the model commits.
- **step35EndpointIgnoresNoFillGroupRect** (I5): dragging a connector endpoint into
  a no-fill group's hollow interior leaves it free (not attached to the group).

PASS line extended with `nofill-border-moves, ink-drag-visible, endpoint-ignores-group`.

## Manual checks (I2/I3 are visual — no automated oracle)

- Create a text box (double-click empty canvas), type; the field is a caret with
  no box/outline. Resize the box → the text scales with it. Change S/M/L/XL while
  editing → the size updates live.
